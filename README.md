# Spire Archive

The complete database for [Slay the Spire](https://store.steampowered.com/app/646570/Slay_the_Spire/) and [Slay the Spire 2](https://store.steampowered.com/app/2868840/Slay_the_Spire_2/) — browse cards, relics, potions, characters, monsters, events, buffs & debuffs, enchantments, and keywords.

**Live site:** [spire-archive.com](https://spire-archive.com)

## What's in here

- **STS1**: 361 cards, 181 relics, 42 potions, 4 characters, and more
- **STS2**: 577 cards, 289 relics, 5 characters, enchantments, and more — parsed from game files
- **CSS card renderer**: Pure HTML/CSS card rendering that matches the in-game look (no pre-rendered images)
- **JSON API**: Filterable endpoints for all game data

## Stack

- **Astro** SSR with `@astrojs/node` adapter
- **Tailwind CSS v4**
- **React islands** for interactive components (card explorer, CSS card renderer, filters)
- Game data in `./data/` (generated by `./parsers/`)

## Local dev

```bash
npm install
npm run dev
# http://localhost:4321
```

## Data pipeline

Parsers in `./parsers/` extract and transform game data:

- `parsers/sts1/parse_all.py` — STS1 data from decompiled game sources
- `parsers/sts2/parse_all.py` — STS2 data from game files
- `parsers/sts2/resolve_vars.py` — resolves template variables in STS2 card descriptions
- `parsers/sts1/resolve_upgrades.py` — computes upgraded card stats
- `parsers/update.sh` — runs the STS2 pipeline

See `parsers/README.md` for the full parser layout.

## API

All endpoints are namespaced by game (`/api/sts1/` and `/api/sts2/`) and return JSON.

| Endpoint | Filters |
|----------|---------|
| `GET /api/{game}/cards` | `q`, `color`, `type`, `rarity`, `cost`, `offset`, `limit` |
| `GET /api/{game}/relics` | `q`, `tier`, `color`, `offset`, `limit` |
| `GET /api/{game}/potions` | `q`, `rarity`, `offset`, `limit` |
| `GET /api/{game}/monsters` | `q`, `act`, `type`, `offset`, `limit` |
| `GET /api/{game}/events` | `q`, `act`, `offset`, `limit` |
| `GET /api/{game}/effects` | `q`, `type`, `offset`, `limit` |
| `GET /api/{game}/search` | `q`, `limit` |

STS2 also has: `/api/sts2/enchantments`

Detail endpoints: `GET /api/{game}/{resource}/:id`
