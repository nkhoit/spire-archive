# STS1 Data

A dark-themed Astro (SSR) site + JSON API for browsing Slay the Spire (STS1) game data.

- **Astro SSR** (`output: server`) with `@astrojs/node` adapter
- **Tailwind CSS v4**
- **React islands** only where needed (client-side filters)
- Data lives in `./data/` (generated by `./parsers/`)

## Local dev

Requirements:
- Node.js 22+

```bash
npm install
npm run dev
```

Site:
- http://localhost:4321

## Build

```bash
npm run build
npm run preview
```

## Docker

```bash
docker build -t sts1-data .
docker run --rm -p 4321:4321 sts1-data
```

## API

All endpoints return JSON with:
- `Cache-Control: public, max-age=3600`

### Cards
- `GET /api/cards?q=&color=&type=&rarity=&cost=&offset=&limit=`
- `GET /api/cards/:id`

### Relics
- `GET /api/relics?q=&tier=&color=&offset=&limit=`
- `GET /api/relics/:id`

### Potions
- `GET /api/potions?q=&rarity=&offset=&limit=`
- `GET /api/potions/:id`

### Monsters
- `GET /api/monsters?q=&act=&type=&offset=&limit=`
- `GET /api/monsters/:id`

### Events
- `GET /api/events?q=&act=&offset=&limit=`
- `GET /api/events/:id`

### Powers
- `GET /api/powers?q=&type=&offset=&limit=`

### Global search
- `GET /api/search?q=&limit=`

### Reference
- `GET /api/reference/keywords`
- `GET /api/reference/orbs`
- `GET /api/reference/stances`
- `GET /api/reference/blights`
- `GET /api/reference/characters`
- `GET /api/reference/achievements`

## Parsers

Parsers remain in `./parsers/` and can generate the JSON files in `./data/`.

Notes:
- `DEFAULT_SOURCE_DIR` / `DEFAULT_LOCALIZATION_DIR` in `parsers/utils.py` are now **relative defaults** (override via CLI args if your paths differ).

## Azure deployment (Container Apps)

Files live in `./infra/`:
- `infra/main.bicep` provisions: ACR (Basic), Container Apps Environment, Container App (external ingress, port 4321, 0–1 replicas)
- `infra/deploy.sh` deploys infra + builds/pushes image + updates the app

Requirements:
- Azure CLI (`az`)
- `docker`
- `jq`

```bash
cd infra
chmod +x deploy.sh

# optional overrides
export LOCATION=westus2
export APP_NAME=sts1-data
export RG=sts1-data-rg

./deploy.sh
```
